# 敏感内容过滤上线验收清单

## 角色矩阵

使用同一资料和同一问题分别验证普通员工、项目成员、商务人员、管理层及系统管理员。普通角色默认无权查看；项目和商务角色依据“系统管理 - 敏感内容管理 - 角色敏感权限”实际配置验收；管理员应能查看全部启用类型。

## 必测内容

- 报价：`本项目报价为 1350 万元。`
- 成本：`设备采购成本约 820 万元。`
- 利润率：`预计毛利率为 18%。`
- 合同金额：`合同总价为 2300 万元。`
- 付款条件：`30% 预付款，70% 验收后支付。`
- 供应商报价：`供应商报价为 500 万元。`

每项均需验证无权限时出现对应隐藏文案、有权限时保持原文。

## 表格语料

- Markdown 表格：单价、总价列隐藏，供应商报价优先按 `supplier_price` 隐藏。
- 横向指标表：第一列为报价、成本、毛利率时，仅隐藏对应值单元格。
- HTML、CSV、TSV、多空格分隔表格：敏感列隐藏且表格仍可读。
- 合同付款表：付款条件列隐藏。
- 工艺参数表：硫酸浓度 20%、镍回收率 98%、反应温度 80℃保持原文。
- 高风险标题或至少 3 个敏感列：无对应权限时整表隐藏。
- 高风险表格解析失败或超过 50,000 字符/500 行：不得放行原始敏感明细。
- 有对应类型权限：相关列或行保持原文；其他未授权类型仍继续过滤。

## 链路检查

分别检查项目问答、基础问答、流式与非流式回答、引用来源、历史消息重新打开和数据库中的 assistant message/citation。任何用户可见位置均不得出现无权限原文。流式接口必须只在服务端完成兜底过滤后发送答案内容。

自动化测试还需断言表格 evidence 在进入 AnswerGenerator 前已经安全化、citation 复用安全内容、最终答案文本兜底仍生效。

## 防误伤样例

以下文本必须保持不变：硫酸浓度 20%、镍回收率 98%、反应温度 80℃、液固比 5:1、处理量 2000 t/a、物料粒径 5mm。

## 失败策略

数据库规则或权限加载异常时，请求应失败并返回统一错误，不得放行原始证据。无数据库的降级/测试链路使用内置保守规则。审计记录只能包含类型、次数、角色和最终答案是否触发兜底过滤，禁止保存敏感原文。

## 自动化结果

执行 `pytest tests/test_sensitive_content_filter.py tests/test_table_sensitive_filter.py tests/test_sensitive_content_acceptance.py`，覆盖列级、行级、格式解析、高风险整表隐藏、角色放行、AnswerGenerator 前置过滤、citation 和最终答案兜底。实际通过数量以本次发布验收命令输出为准。
