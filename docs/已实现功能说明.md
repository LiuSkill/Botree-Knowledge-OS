# Botree Knowledge 已实现功能说明

> 基于当前仓库代码梳理，范围覆盖 `backend/`、`frontend/`、主要接口定义、核心 Service 逻辑与现有测试用例。  
> 梳理时间：2026-06-16

## 1. 系统定位与整体架构

本系统已经落地为一个“企业知识 + 项目知识 + 文档评审 + AI 问答”的一体化平台，目标是把资料沉淀、审核发布、索引构建和问答检索串成闭环。

### 1.1 前端技术栈

- `Vue 3 + TypeScript + Vite`
- `Pinia` 负责登录态与权限状态管理
- `Vue Router` 负责路由和权限守卫
- `TDesign Vue Next` 负责 UI 组件
- `Axios` 负责接口请求
- `Markdown-It` 负责文档预览区 Markdown 渲染

### 1.2 后端技术栈

- `FastAPI` 提供 REST API
- `SQLAlchemy + Alembic` 管理数据模型与迁移
- `PyMySQL` 连接关系型数据库
- `Redis + RQ` 支撑异步任务
- `MinIO` 用于文档及预览资源存储
- `Milvus` 用于向量检索
- 图谱检索、页索引检索、关键词检索已经在代码中集成到统一检索路由

### 1.3 当前系统已形成的业务分层

后端基本遵循 `Controller -> Service -> Repository -> Database` 的分层方式：

- `API Router` 负责路由、入参和权限注入
- `Service` 负责核心业务规则
- `Repository` 负责查询和持久化
- `Database / Storage / Index` 负责数据、文件和索引资源

## 2. 已落地的核心业务闭环

当前代码已实现以下端到端流程，且关键流程已有测试覆盖：

1. 用户登录后进入工作台，根据角色与权限访问不同模块。
2. 创建项目时，系统自动创建对应的项目知识库，并自动初始化默认项目分类树。
3. 用户向企业知识库或项目知识库上传资料，系统自动生成文档记录和 `V1` 版本。
4. 上传后的文档先进入草稿或待审核状态，并自动创建解析任务。
5. 文档可提交审核，审核人可通过或驳回。
6. 审核通过后，文档可执行解析、质检、索引构建与发布。
7. 文档支持上传新版本，新版本不会直接替换现网生效版本，必须重新走审核与索引流程。
8. 新版本索引成功后，系统自动切换当前生效版本，并使旧版本转为历史版本。
9. AI 问答可基于企业知识或项目知识进行检索，并返回引用、溯源信息和执行轨迹。
10. 删除文档时，数据库记录、页索引、图谱、引用关系、检索追踪等会一并清理，外部存储和向量索引采用后台异步清理。

## 4. 后端已实现功能

## 4.1 认证、用户、角色与权限

### 认证与登录

- 支持用户名密码登录。
- 登录时校验账号状态是否启用。
- 登录成功后签发 JWT。
- 可获取当前登录用户信息，包括角色和权限编码。
- 登录、退出等关键动作已接入操作日志。

### 用户管理

- 支持用户新增、查询、修改、删除。
- 支持重置密码接口。

### 角色与权限

- 支持角色新增、查询、修改、删除。
- 支持权限矩阵查询，供前端权限矩阵页面展示。

## 4.2 项目与项目知识库

- 支持项目新增、查询、详情、修改、删除。
- 支持按当前用户权限过滤项目可见范围。
- 创建项目时自动：
  - 把创建人加入项目成员并赋予项目管理范围。
  - 自动创建对应项目知识库。
  - 自动初始化默认项目分类树。
- 支持项目成员列表、成员新增、成员移除接口。

默认创建的项目分类已落地为三大类：

- 设计资料：设计输入、设计计算、设计评审
- 实施资料：会议纪要、现场记录、变更资料
- 交付资料：验收资料、运维手册、归档文件

## 4.3 知识库与分类

### 知识库

- 支持企业知识库与项目知识库两种类型。
- 支持知识库新增、列表、详情、修改、删除。
- 支持知识库授权摘要接口。
- 返回知识库下的文档数量与分块数量。

### 知识分类

- 支持企业级分类树和项目级分类树。
- 支持分类新增、编辑、删除、树形查询。
- 支持多级嵌套分类。
- 删除分类时会校验是否存在子分类或关联文档。
- 支持文档上传前的分类合法性校验。

## 4.4 文档、版本、解析与索引

### 文档上传与版本管理

- 支持上传文档到企业知识库或项目知识库。
- 首次上传自动创建文档主记录与 `V1` 版本记录。
- 支持新版本上传。
- 新版本上传后不会立即替换当前生效版本。
- 支持版本列表、版本下载、版本审核流转。
- 支持文档归档。
- 支持文档回滚接口。

### 文档状态流转

代码中已落地的核心状态包括：

- 审核状态：`draft`、`reviewing`、`approved`、`rejected`、`archived`
- 文档状态：`pending_review`、`reviewed`、`active`、`inactive`、`archived`
- 解析状态：`unparsed`、`parsing`、`success`、`failed`
- 索引状态：`not_indexed`、`parsing`、`parsed`、`parsed_pending_review`、`indexing`、`indexed`、`failed`、`invalid`
- 版本状态：`pending_review`、`draft`、`approved`、`current`、`historical`、`inactive`、`rejected`

### 解析与页级处理

- 上传文档后自动创建解析任务。
- 支持指定版本解析，不影响当前生效版本切换。
- 已支持 Office 文档先转 PDF，再接入解析流程。
- 已支持文档页、页面块、页面索引、文本镜像落库。
- 已支持页级质检。
- 已支持页级人工修正接口。
- 已支持页面图纸号等辅助信息提取与索引。

### 索引构建与发布

- 支持文档同步索引构建。
- 支持文档异步索引构建任务。
- 支持版本级索引构建。
- 支持页索引、关键词检索文本镜像、向量索引、图谱索引的统一构建。
- 索引构建成功后自动发布页索引和图谱数据。
- 新版本索引成功后会自动切换当前版本并失效旧版本索引。
- 若新版本构建失败，旧版本保持继续生效。
- 同一文档存在活动中的全量构建任务时，会阻止重复发起。

### 文档预览与资源访问

- 支持文档详情查询。
- 支持分页查询文档列表。
- 支持文档分块查询。
- 支持文档页面查询。
- 支持预览结果输出：
  - Markdown 内容
  - 转换后的 PDF 资源
  - 页级预览图
  - 页面块图片资源
- 支持文档附件和预览资源下载接口。

### 文档删除清理

- 删除文档时会清理：
  - 文档主记录
  - 版本记录
  - 分块
  - 页面与页面块
  - 页索引
  - 图谱实体与关系
  - 审核任务与审核日志
  - 聊天引用
  - 检索追踪
  - 索引任务
- 外部对象存储和向量索引清理采用后台异步任务，避免阻塞删除接口。

## 4.5 审核流

- 支持提交审核。
- 提交审核时可针对当前草稿版本或指定新版本。
- 审核通过后会更新任务状态、版本状态以及文档主状态。
- 审核驳回后会保留版本记录，并允许继续修订或再次提审。
- 审核日志已落库。
- 已支持查询审核任务列表、审核详情、审核通过文档列表。

## 4.6 检索与 AI 问答

### 检索能力

- 支持统一检索入口。
- 支持检索调试接口。
- 检索结果会被转换为可引用的 citation 数据结构。
- 代码中已接入多种检索手段组合：
  - 关键词检索
  - 页索引检索
  - 向量检索
  - 图谱检索

### 视觉证据增强

- 对包含图号、图片、页面定位等特征的问题，会优先补充可视化证据。
- 引用中可附带页预览图或块级图片资源。
- 资源通过安全接口对外暴露，不直接透出底层存储路径。

### 会话与问答

- 支持企业知识问答和项目知识问答两种模式。
- 支持会话创建、会话列表、消息历史、会话删除。
- 项目问答必须指定项目上下文并校验访问权限。
- 企业知识问答会限制外部用户访问。
- 问答结果支持返回：
  - 最终答案
  - 会话 ID
  - 问答类型
  - 检索范围
  - 使用的检索器
  - Agent 轨迹
  - Trace Steps
  - 引用列表
  - 调试原始信息

### 轨迹与审计

- 支持保存问答检索轨迹。
- 轨迹内容包括意图、子查询、召回结果、重排结果、引用结果、耗时等。
- 支持按消息查看问答轨迹。
- 问答行为会写入操作日志与 QA 审计数据。

## 4.7 模型配置与系统能力

### 模型配置

- 支持模型配置新增、查询、修改、删除。
- 支持设置默认模型。
- 支持模型联通性测试。

  - `llm`
  - `vision_llm`
  - `embedding`
  - `reranker`
  - `intent`
  - `evidence_judge`
  - `graph_extractor`

### 系统能力

- 支持仪表盘统计接口。
- 支持操作日志查询。
- 支持 QA 审计记录查询。
- 支持检索追踪列表查询。
- 支持健康检查接口。

## 6. 当前已落地与部分实现边界

以下边界建议在对外说明时单独标注，避免误解：

- 授权中心前端当前以“授权概览展示”为主，外部用户精细化授权结构已预留，但未形成完整前台管理流程。
- 项目成员能力后端已支持新增/移除接口，但前端当前主要是成员展示。
- 用户、角色、模型配置后端已支持更完整 CRUD，但前端系统管理页当前主要覆盖“列表/创建/默认设置/查看”这类操作。
- 文档页级人工纠错、页级质检、同步索引等后端能力已存在，但主前端入口当前重点放在“查看、提审、版本、异步构建”。
- 企业知识问答对外部用户有显式限制，这是当前产品规则，不是缺陷。

## 8. 结论

当前系统已经实现了以下可用产品骨架：

- 企业知识和项目知识双知识域管理
- 文档上传、版本管理、审核流转
- 解析、质检、索引构建、发布切换
- 基于多路检索的 AI 问答
- 引用溯源、轨迹审计、操作日志
- 基础用户、角色、模型配置与系统看板

如果后续要继续扩展，最适合优先补齐的方向通常是：

1. 授权中心的精细化配置前端。
2. 项目成员管理前端操作入口。
3. 文档页级纠错、质检、同步构建等后端能力的可视化入口。
4. 系统管理模块的编辑、删除、重置密码等完整运营操作。
