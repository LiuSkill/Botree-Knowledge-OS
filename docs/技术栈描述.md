# 项目技术栈描述

本文基于当前仓库的 `package.json`、`requirements.txt` 与代码实现整理，按前端和后端分类说明项目技术栈。

## 前端

| 技术 | 说明 | 用途 |
| --- | --- | --- |
| Vue 3 | 核心前端框架 | 构建单页应用与业务页面 |
| TypeScript | 主要开发语言 | 提升类型安全与可维护性 |
| Vite | 构建工具与开发服务器 | 本地开发、打包构建、热更新 |
| Vue Router | 路由管理 | 页面路由、登录守卫、动态路由 |
| Pinia | 状态管理 | 管理用户登录态、应用状态等 |
| TDesign Vue Next | UI 组件库 | 提供企业后台常用表单、表格、弹窗等组件 |
| `@tdesign-vue-next/chat` | 聊天组件 | 支撑 AI 对话相关界面 |
| Axios | HTTP 请求库 | 与后端 API 交互 |
| VueUse | 组合式工具库 | 提供常用响应式与浏览器能力封装 |
| markdown-it | Markdown 渲染 | 文档、问答内容等富文本展示 |
| KaTeX | 数学公式渲染 | 支持公式内容展示 |
| tdesign-icons-vue-next | 图标库 | 页面按钮、导航和操作图标 |
| vue-tsc | 类型检查工具 | 配合构建流程进行类型校验 |

### 前端特征

- 采用 Vue 3 + TypeScript + Vite 的现代化工程体系。
- 以企业后台场景为主，使用 TDesign 统一视觉与交互。
- 通过 Vue Router 和 Pinia 组织页面路由与全局状态。
- 面向知识库、项目、审核、系统管理、AI 对话等业务页面。

## 后端

| 技术 | 说明 | 用途 |
| --- | --- | --- |
| FastAPI | Web 框架 | 提供 REST API 与自动生成接口文档 |
| Uvicorn | ASGI 服务器 | 启动后端服务 |
| Python | 主要开发语言 | 实现业务逻辑、检索、AI 编排等能力 |
| SQLAlchemy | ORM | 数据库访问与对象映射 |
| Pydantic | 数据校验与 Schema | 请求/响应模型定义与参数校验 |
| Pydantic Settings | 配置管理 | 统一读取环境变量与运行配置 |
| PyJWT | JWT 认证 | 登录态签发与鉴权 |
| python-multipart | 文件上传支持 | 处理表单与文件上传请求 |
| MySQL | 关系型数据库 | 存储用户、项目、知识库、审核等核心数据 |
| Redis | 缓存与消息组件 | 支撑任务队列和运行时缓存 |
| RQ | 任务队列 | 异步任务与离线索引构建 |
| MinIO | 对象存储 | 存储上传文档、附件和资源文件 |
| Milvus | 向量数据库 | 存储向量索引，支持语义检索 |
| Alembic | 数据库迁移 | 管理数据库结构演进 |
| langgraph | Agent 编排 | 支撑 AI 问答流程、检索规划与多步推理 |
| sentence-transformers | 向量生成 | 生成文本 Embedding |
| transformers | 模型推理基础库 | 支撑本地/远程模型调用 |
| torch | 深度学习运行时 | 支撑本地模型推理 |
| accelerate | 推理加速 | 优化大模型/嵌入模型执行 |
| requests | HTTP 客户端 | 调用外部服务，如模型和解析接口 |
| python-docx | Word 文档处理 | 解析或生成 DOCX 文档 |
| pypdf | PDF 处理 | 读取和解析 PDF 文件 |
| pymilvus | Milvus 客户端 | 连接向量数据库 |
| 分层架构 | 代码组织方式 | Controller -> Service -> Repository -> Database 分层 |

### 后端特征

- 基于 FastAPI 构建 REST 风格接口，并统一返回 `{ code, message, data }`。
- 采用 `Controller -> Service -> Repository -> Database` 分层，业务逻辑集中在 Service 层。
- 支持登录鉴权、知识库管理、文档上传、审核、检索、AI 问答与系统配置。
- 集成 MySQL、Redis、MinIO、Milvus 等基础设施，覆盖结构化数据、缓存、对象存储和向量检索。
- 通过 LangGraph 和本地/远程模型能力支撑检索增强生成（RAG）和 Agent 流程。

## 总体说明

- 前后端通过 HTTP API 解耦。
- 前端负责交互、路由、状态和页面呈现。
- 后端负责认证、业务规则、数据持久化、检索与 AI 流程编排。
- 项目整体面向企业知识管理、项目协作与 AI 问答场景。
