# 快速财务计算器计算说明

本文档说明快速财务计算器 MVP 的数据补齐口径、计算流程和各项指标来源。当前计算器不执行 Excel 原公式，也不硬编码静态假数据；所有结果均来自后台配置表中的结构化字段。

### 2. 后续遇到提示时怎么补

| 提示类型 | 本质原因 | 补齐位置 | 补齐方式 |
| --- | --- | --- | --- |
| `未配置收率，当前仅按产出系数计算` | 路线产出配置有 `output_ratio`，但 `recovery_rate <= 0` | 工艺路线库 -> 路线详情 -> 路线测算产出；表：`process_calculation_outputs` | 如果产出系数已包含收率，填 `1.000000`；如果需单独折减，填实际收率，如 `0.95`。 |
| `表达式没有可用的结构化数值` | 配置中只有 Excel 表达式或表达式未解析，结构化系数为 0 | 工艺节点库 -> 消耗品/公共服务/三废产出；表：`process_node_consumables`、`process_node_public_services`、`process_node_outputs` | 计算出固定系数，写入 `amount_per_ton_bm` 或 `output_per_ton`；必要时把 `formula_type` 改为 `fixed`。 |
| `未配置区域有效单价` | 对应原料、产品、消耗品、公辅没有当前区域价格 | 原料库/产品库/消耗品库/公共服务库的区域价格；表：`process_region_prices` | 按 `owner_type + owner_id + region_code` 补单价、币种、单位并启用。 |
| CAPEX 为 `0` | 当前路线节点没有设备投资，或设备币种与测算币种不一致 | 工艺节点库 -> 设备投资；表：`process_node_equipment` | 按节点补 `investment_amount` 和 `currency`，币种需与区域币种一致。 |
| IRR 为 `-` | 现金流没有同时出现负值和正值，数学上无法求解内部收益率 | 由 CAPEX、OPEX、收入共同决定 | 通常需要补 CAPEX 和有效收入/成本，不能直接填 IRR。 |
| 回收期为 `0.00 年` | CAPEX 为 0 时，初始现金流不需要回收 | 由 CAPEX 和现金流共同决定 | 需要补节点设备投资后才会形成实际回收期。 |

## 二、输入参数

快速财务计算器接口入参来自前端测算面板：

| 字段 | 含义 | 计算用途 |
| --- | --- | --- |
| `materials[].material_id` | 原料 ID | 匹配路线、读取原料价格、确定节点处理量。 |
| `materials[].amount` | 年处理量 | 产品产出、消耗、三废、物料平衡、CAPEX 规模修正的基础量。 |
| `materials[].unit` | 原料单位 | 统一换算为 `t`。 |
| `target_products[]` | 目标产品 ID | 匹配能产出目标产品的启用路线。 |
| `region_code` | 区域：`asia/europe/americas` | 读取区域价格。 |
| `currency` | 币种：`CNY/EUR/USD` | 与区域绑定，当前不做汇率换算。 |
| `tax_rate` | 所得税率 | 年度现金流中计算税费。 |
| `discount_rate` | 折现率 | NPV 和折现回收期。 |
| `period_years` | 测算周期 | 年度现金流年限。 |
| `sort_criteria` | 推荐排序指标 | 对路线组合排序，如 NPV、IRR、EBITDA。 |
| `advanced_params.base_capacity` | 基准产能 | 与 `scale_param_n` 一起用于 CAPEX 规模修正。 |
| `advanced_params.scale_param_n` | 规模指数 n | CAPEX 修正指数。 |
| `advanced_params.other_opex` | 其他年度 OPEX | 直接计入 OPEX。 |
| `advanced_params.annual_growth_rate` | 年增长率 | 每年收入和 OPEX 同比例增长。 |

## 四、路线匹配与组合

### 1. 一对一路线匹配

每条工艺路线表达一个“输入原料 -> 一个最终产品”的完整线路。

匹配条件：

```text
route.input_material_id in 输入原料
route.final_product_id in 目标产品
route.status = enabled
route.is_deleted = false
```

### 2. 一对多目标产品组合

如果输入一个原料、选择多个目标产品，系统会先分别找出每个目标产品的候选路线，再做组合。

示例：

```text
黑粉BM -> 电池级硫酸钴：候选路线 A、B
黑粉BM -> 电池级硫酸镍：候选路线 C、D

组合结果：
A + C
A + D
B + C
B + D
```

当前 MVP 限制：

```text
每个原料/产品组合最多取 5 条候选路线
最多计算 100 个路线组合
最终返回排序后的前 3 个方案
```

### 3. 共享节点去重

同一个路线组合中，多个目标产品可能共用前段节点。计算节点消耗、三废和 CAPEX 时，会按节点 ID 去重，避免同一个共享节点重复计费。

节点处理量计算：

```text
node_amount_t = sum(经过该节点的输入原料年处理量)
```

如果同一原料下多条目标产品路线共用同一节点，该节点只计一次该原料处理量。

## 五、产品产出与收入

### 1. 产品产出量

优先读取路线维度产出配置 `process_calculation_outputs`，只计算：

```text
output_type in ("product", "byproduct")
```

公式：

```text
product_amount = route_material_amount_t * output_ratio * recovery_rate
```

字段来源：

```text
route_material_amount_t：该路线输入原料年处理量，统一为 t
output_ratio：process_calculation_outputs.output_ratio
recovery_rate：process_calculation_outputs.recovery_rate
```

如果 `recovery_rate <= 0`，当前服务会临时按 `1` 计算，并产生提示。推荐显式配置为 `1.000000` 或实际收率。

### 2. 产品收入

公式：

```text
product_revenue = converted_product_amount * product_region_unit_price
revenue = sum(product_revenue)
```

字段来源：

```text
product_region_unit_price：process_region_prices.unit_price
price owner_type：product
region_code：测算选择区域
currency：必须等于测算币种
unit：必须能与产出单位换算
```

如果单价币种与测算币种不一致，当前 MVP 不做汇率换算，该项金额按 0 处理并提示。

## 六、药剂成本

### 1. 药剂消耗量

节点药剂配置来自 `process_node_consumables`。

系数选择规则：

```text
coefficient = amount_per_ton_bm if amount_per_ton_bm != 0 else amount_per_ton
```

公式：

```text
consumable_amount = node_amount_t * coefficient
```

单位处理：

```text
kg/t-BM -> kg
t/t-BM -> t
KWH/t-BM -> KWH
```

### 2. 药剂成本

公式：

```text
consumable_cost_item = converted_consumable_amount * consumable_region_unit_price
consumable_cost = sum(consumable_cost_item)
```

字段来源：

```text
consumable_region_unit_price：process_region_prices.unit_price
price owner_type：consumable
```

## 七、公辅成本

公辅成本与药剂成本同口径，配置来自 `process_node_public_services`。

公式：

```text
public_service_amount = node_amount_t * coefficient
public_service_cost_item = converted_public_service_amount * public_service_region_unit_price
public_service_cost = sum(public_service_cost_item)
```

字段来源：

```text
coefficient：amount_per_ton_bm 或 amount_per_ton
unit：process_node_public_services.unit
unit_price：process_region_prices.unit_price
price owner_type：public_service
```

## 八、三废产出与处理成本

三废来自节点维度配置 `process_node_outputs`，只计算：

```text
output_type in ("solid_waste", "wastewater")
```

### 1. 三废产出量

公式：

```text
waste_amount = node_amount_t * output_per_ton
```

字段来源：

```text
output_per_ton：process_node_outputs.output_per_ton
unit：process_node_outputs.unit
```

### 2. 三废处理成本

处理单价优先级：

```text
1. process_region_prices 中 owner_type=product 的区域价格
2. process_node_outputs.treatment_cost
3. process_products.treatment_cost
```

公式：

```text
waste_treatment_cost_item = converted_waste_amount * treatment_unit_price
waste_treatment_cost = sum(waste_treatment_cost_item)
```

## 九、原料成本

原料成本按输入原料去重计算，不按目标产品重复计。

公式：

```text
material_cost_item = material_amount_t * material_region_unit_price
material_cost = sum(material_cost_item)
```

字段来源：

```text
material_amount_t：输入原料年处理量，统一换算为 t
material_region_unit_price：process_region_prices.unit_price
price owner_type：material
```

## 十、CAPEX

CAPEX 来自路线组合中去重后的节点设备投资配置 `process_node_equipment`。

基础公式：

```text
capex = sum(node_equipment.investment_amount)
```

币种规则：

```text
equipment.currency 必须等于测算 currency
否则该设备投资不计入 CAPEX，并产生提示
```

规模修正：

```text
如果 base_capacity 和 scale_param_n 同时填写：
capex = capex * (actual_capacity / base_capacity) ^ scale_param_n
```

字段说明：

```text
actual_capacity = sum(输入原料年处理量t)
base_capacity = advanced_params.base_capacity
scale_param_n = advanced_params.scale_param_n
```

如果节点没有设备投资配置，CAPEX 就是 `0`。这会进一步导致回收期显示 `0.00 年`，IRR 可能无法计算。

## 十一、OPEX

年度 OPEX 由五部分组成：

```text
opex = material_cost
     + consumable_cost
     + public_service_cost
     + waste_treatment_cost
     + other_opex
```

字段来源：

```text
material_cost：原料成本
consumable_cost：药剂成本
public_service_cost：公辅成本
waste_treatment_cost：三废处理成本
other_opex：advanced_params.other_opex
```

## 十二、EBITDA

公式：

```text
ebitda = revenue - opex
```

当前 MVP 未单独计算折旧、摊销和财务费用，因此 EBITDA 直接使用收入减年度 OPEX。

## 十三、年度现金流

现金流从第 0 年到第 `period_years` 年。

### 1. 第 0 年

```text
year0_net_cash_flow = -capex
year0_discounted_cash_flow = -capex
```

### 2. 第 1 年至第 N 年

增长系数：

```text
growth_factor(year 1) = 1
growth_factor(year n) = (1 + annual_growth_rate) ^ (n - 1)
```

年度收入和成本：

```text
annual_revenue = revenue * growth_factor
annual_opex = opex * growth_factor
annual_ebitda = annual_revenue - annual_opex
```

所得税：

```text
tax = max(annual_ebitda, 0) * tax_rate
```

净现金流：

```text
net_cash_flow = annual_ebitda - tax
```

折现现金流：

```text
discounted_cash_flow = net_cash_flow / (1 + discount_rate) ^ year
```

## 十四、NPV

公式：

```text
npv = sum(discounted_cash_flow from year 0 to period_years)
```

也就是：

```text
npv = -capex
    + sum(net_cash_flow_year_n / (1 + discount_rate)^n)
```

当前页面上的 `NPV（3年）` 或 `NPV（10年）` 由 `period_years` 决定。

## 十五、IRR

IRR 是让 NPV 等于 0 的折现率：

```text
0 = sum(cash_flow_year_n / (1 + irr)^n)
```

当前服务用二分法求解，范围从 `-99.99%` 起步向上扩展。

如果现金流序列没有同时出现负值和正值，则 IRR 数学上不可解，页面显示 `-`。

常见原因：

```text
CAPEX = 0，所有现金流非负
或项目持续亏损，所有现金流非正
```

## 十六、回收期

回收期按未折现现金流累计计算。

公式：

```text
cumulative_cash_flow(year n) = sum(net_cash_flow from year 0 to year n)
```

首次累计现金流大于等于 0 的年份为回收年份。

如果跨年回收，则用线性插值：

```text
payback_period = previous_year + abs(previous_cumulative_cash_flow) / current_year_cash_flow
```

如果第 0 年现金流已经大于等于 0，回收期为 `0`。这通常意味着 CAPEX 为 `0`。

## 十七、折现回收期

折现回收期与回收期相同，但累计对象换成 `discounted_cash_flow`。

```text
discounted_payback_period = payback(discounted_cash_flows)
```

当前接口已计算该值，前端摘要主要展示静态回收期。

## 十八、物料平衡

物料平衡只统计能换算为质量 `t` 的产出物。

公式：

```text
input_mass_t = sum(输入原料年处理量t，按原料去重)
accounted_output_mass_t = sum(产品产出 + 三废产出中可换算为t的数量)
difference_mass_t = input_mass_t - accounted_output_mass_t
balance_rate = accounted_output_mass_t / input_mass_t
```

如果某些产出单位不能换算为质量，比如 `m3`、`KWH`，会进入排除清单并产生提示。

## 十九、路线推荐排序

系统先计算所有候选组合，再按 `sort_criteria` 排序，并返回前三条。

排序前置规则：

```text
1. 配置完整的路线优先于有提示的路线
2. 提示数量少的路线优先
3. 再按用户选择的指标排序
```

指标方向：

| 指标 | 排序方向 |
| --- | --- |
| `npv` | 越高越好 |
| `irr` | 越高越好 |
| `ebitda` | 越高越好 |
| `payback_period` | 越短越好 |
| `capex` | 越低越好 |

## 二十、输出清单

接口返回的核心结果如下：

| 输出字段 | 含义 | 计算来源 |
| --- | --- | --- |
| `matched_routes` | 排序后的候选路线方案 | 路线匹配 + 组合 + 指标排序 |
| `recommended_route` | 推荐路线 | `matched_routes[0]` |
| `product_outputs` | 推荐路线产品/副产物产出 | 路线测算产出配置 |
| `consumable_costs` | 推荐路线药剂消耗和成本 | 节点消耗品配置 + 区域价格 |
| `public_service_costs` | 推荐路线公辅消耗和成本 | 节点公辅配置 + 区域价格 |
| `waste_outputs` | 推荐路线三废产出和处理成本 | 节点三废配置 + 处理单价 |
| `capex` | 总投资 | 去重节点设备投资，必要时做规模修正 |
| `opex` | 年度运营成本 | 原料 + 药剂 + 公辅 + 三废 + 其他 OPEX |
| `revenue` | 年度营业收入 | 产品产出 * 产品区域价格 |
| `ebitda` | 年度 EBITDA | `revenue - opex` |
| `npv` | 净现值 | 折现现金流合计 |
| `irr` | 内部收益率 | 使 NPV 为 0 的折现率 |
| `payback_period` | 静态回收期 | 未折现现金流累计回正时间 |
| `material_balance` | 物料平衡 | 输入质量与可计量产出质量对比 |
| `cash_flows` | 年度现金流表 | 第 0 年 CAPEX + 每年收入/OPEX/税/净现金流 |
| `warnings` | 配置提示 | 缺系数、缺价格、币种不一致、单位不可换算等 |

## 二十一、当前 MVP 不做的事项

当前计算器为了保持配置可控和结果可解释，暂不做以下事情：

```text
不执行 Excel 原公式
不把公式硬编码进代码
不做跨币种汇率换算
不自动推导所有元素收率
不自动生成设备投资
不把三废当成目标产品参与路线匹配
不对无法换算单位的物料强行计入物料平衡
```

这些能力后续可以在“公式引擎、汇率表、CAPEX 模板、元素平衡模型”稳定后逐步增强。
