# 敏感内容过滤上线前验收报告

验收日期：2026-07-20

## 验收结论

敏感内容过滤核心链路、管理接口、前端管理页、权限控制、流式输出及审计字段通过专项验收。验收中发现并修复了最终答案在兜底脱敏前进入 trace/日志、问题和证据原文进入日志，以及管理页缺少缓存刷新和审计查询入口的问题。

建议进入小范围灰度试用。灰度前应在目标环境备份数据库并执行迁移，灰度期间重点观察过滤命中率、误伤率、规则加载异常和最终答案兜底命中指标。

## 已通过项

- 数据库迁移、ORM、Repository、Service、API、问答链路和前端字段保持一致；目标迁移已在模拟现有基线库上成功升级。
- 报价、成本、利润率、合同金额、付款条件、供应商报价、商务策略、财务指标均有规则及测试覆盖。
- 多角色权限按并集计算；管理员可查看全部已启用敏感类型。
- 工艺浓度、回收率、温度、液固比、处理量和粒径未被误伤。
- 检索证据在进入回答模型和 citation 前脱敏，最终答案在进入 trace、日志、持久化和响应前执行兜底过滤。
- 流式回答先缓冲完整模型输出，完成兜底过滤后只发送一次安全 delta，不提前发送原始 token。
- 规则或角色权限加载异常会向上抛出并阻断回答，不回退放行原文。
- 所有敏感内容管理 API 均使用管理员依赖；普通用户访问返回 403。
- 应用响应源码中不存在 `raw_answer`、`raw_chunk_content`、`raw_snippet`、`unmasked_content` 字段。
- 脱敏审计只记录用户/角色/会话定位信息、敏感类型、次数和 `final_answer_redacted`，不记录敏感原文。
- 管理页支持敏感类型和规则的查询、新增、修改、启停，规则测试，角色权限矩阵保存，缓存刷新和审计查询。
- 前端 `npx vue-tsc --noEmit` 通过；`npm run build` 通过。
- 敏感内容专项及受影响回归测试 39 项通过。

## 本次发现并修复的问题

1. 最终答案原先在兜底脱敏前生成回答 trace，并写入 `answer_preview` 日志。现已调整为先过滤，再生成 trace、日志和响应。
2. 问题原文、查询改写及证据 `content_preview` 原先可能进入运行日志。现改为不可逆哈希、长度、数量和来源元数据，不记录正文。
3. 管理页原先未接入缓存刷新和审计查询。现已补齐 API 类型、操作按钮、审计表格和分页查询。
4. 缺少合同金额、付款条件、角色权限、管理员、异常阻断、日志安全、接口 403、最终答案和流式缓冲专项测试。现已补齐。

## 未阻断灰度的问题

- 全量后端测试结果为 329 通过、2 失败。失败项与敏感内容功能无关：一项受本机 `.env` 数据库配置影响，另一项是 Windows 下 LibreOffice `file:///C:/...` 路径断言差异。
- 前端构建仍有既有的大分包体积警告，不影响类型检查和构建产物生成。
- 本次验收完成了代码、接口依赖、自动化测试、迁移和生产构建检查；目标环境仍需按发布流程执行一次管理员页面人工冒烟和真实 SSE 代理链路冒烟。

## 灰度建议

建议进入小范围灰度，不建议直接全量。灰度准入条件：目标库迁移成功、管理员页面人工冒烟通过、普通账号确认 403、至少使用一个无敏感权限角色和一个管理员角色完成真实问答对照。若出现规则/权限加载异常，应保持当前失败关闭策略并停止扩大流量。
